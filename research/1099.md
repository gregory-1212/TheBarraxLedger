# 1099 Tracking — Market Research

> Generated 2026-05-22 by a research agent surveying 1099 contractor tracking + filing tools. See [README.md](README.md) for the synthesis decisions distilled from this research.

## Summary

For a 2-user shop with 3-8 contractors, the market splits into three useful tiers: (1) **per-form filing services** (Track1099, Tax1099, TaxBandits) that excel at year-end W-9 + e-file, (2) **bundled payroll/contractor platforms** (Gusto, Deel, Rippling) that pay contractors and file 1099s as a side effect, and (3) **the IRS IRIS portal**, which makes free direct e-file genuinely viable in 2026. For The Barrax's profile — payments already flow through other rails, CPA hand-off is the real deliverable — the cheapest, lowest-surface-area path is to build internal W-9 + YTD tracking and let either IRIS (free) or Track1099 (~$3.10/form) handle the actual filing.

## Solutions Reviewed

### 1. Track1099 (Avalara)
- **What it is:** The de-facto standard 1099 e-file service for SMBs and bookkeepers; acquired by Avalara. Strong W-9 collection workflow, IRS + state e-file, recipient delivery built-in.
- **Pricing:** Pay-as-you-go tiers — $3.10/form for the first 15, $2.30 for 16-165, then drops sharply. State e-file $1.49/form. TIN matching $0.45/form. Postal mail $1.84-$2.89. Recipient e-delivery included.
- **W-9 collection:** Enter vendor name + email, vendor receives a link, fills out W-9 in browser, data lands in dashboard. CSV import for bulk. Embeddable widget + API for enterprise.
- **Tax ID security:** TINs are encrypted; the W-9 collection happens on Avalara's secured portal so the SSN/EIN never lives in your email. TIN matching against IRS records is a paid add-on.
- **YTD tracking:** Does not track payments natively — expects you to import totals from QuickBooks, Xero, or CSV at year-end.
- **E-file support:** Full IRS + state e-file, including CF/SF states. Files corrections too.
- **UI patterns to steal:** "Send W-9 request" as a one-click action from the contractor row; status pill (Requested / Received / Verified); year-end "ready to file" checklist that flags missing TINs.
- **Gaps:** Not a payment system. Per-form pricing is fine at our scale but adds up fast over 20+ contractors.

### 2. Tax1099.com
- **What it is:** Direct competitor to Track1099, slightly cheaper at low volumes, IRS-authorized e-file provider, same W-9 request flow.
- **Pricing:** $3.10/form for first 15 (identical to Track1099). TIN matching $1/each real-time. Print & mail $1.90. e-Delivery $0.25/form (not bundled like Track1099).
- **W-9 collection:** Electronic W-9/W-8 request via email link; bulk CSV import. Has a TIN-matching workflow before filing.
- **Tax ID security:** SOC 2 / 256-bit encryption claimed. TINs masked in UI.
- **YTD tracking:** Same as Track1099 — import-only.
- **E-file support:** IRS + state, CF/SF supported.
- **UI patterns to steal:** "Validate TIN before file" prompt that blocks submission if mismatched. Per-form fee shown live as you build the batch.
- **Gaps:** UI feels older than Track1099; e-delivery costs extra.

### 3. TaxBandits
- **What it is:** IRS-authorized filer with a notably generous free W-9 manager — you can request, collect, and store W-9s without paying anything until you file.
- **Pricing:** ~$2.75/form federal, $0.70/form state at low volumes; free W-9 collection and storage year-round.
- **W-9 collection:** Free unlimited W-9 requests + storage; recipient gets an email, completes in browser, payer gets PDF. Bulk TIN match available.
- **Tax ID security:** SOC 2 Type II; TINs masked, audit trail.
- **YTD tracking:** Not native; manual or CSV import.
- **E-file support:** IRS + state + recipient mail.
- **UI patterns to steal:** Free-tier W-9 vault as a standalone product — implies the W-9 collection step is what people want most.
- **Gaps:** Brand recognition lower than Track1099; CPAs less familiar with the format.

### 4. Gusto Contractor-Only Plan
- **What it is:** Payroll-grade ACH payment + automatic year-end 1099-NEC filing. Contractors self-onboard, including W-9 data.
- **Pricing:** $35/mo + $6/contractor/mo. For 5 contractors: $65/mo = $780/yr. 1099 prep + filing + recipient delivery included.
- **W-9 collection:** Contractor self-onboards via emailed invite, enters TIN and address themselves.
- **Tax ID security:** Bank-grade; TIN never visible after entry.
- **YTD tracking:** Tracks every payment made through Gusto automatically. Doesn't see payments made outside (e.g., manual checks).
- **E-file support:** Files with IRS + state; mails/emails recipient copies.
- **UI patterns to steal:** Self-onboarding link is the gold standard — payer never touches the SSN. Year-end "1099s are filed" notification.
- **Gaps:** Only worthwhile if you actually pay contractors *through* Gusto. At ~$780/yr for the privilege, overkill if contractor payments already flow through bank ACH or Stripe.

### 5. IRS IRIS (free direct e-file)
- **What it is:** IRS-operated portal launched 2023, fully GA in 2026; will be the *only* IRS intake for filing season 2027 (replaces legacy FIRE). Free e-filing of all 1099 variants.
- **Pricing:** $0.
- **W-9 collection:** Not provided — IRIS is intake-only.
- **Tax ID security:** Filer's responsibility before submission; IRIS auths via ID.me + 5-digit TCC.
- **YTD tracking:** Not provided.
- **E-file support:** Manual entry or CSV upload using IRS-provided template; download payee copies to distribute yourself.
- **UI patterns to steal:** The IRS CSV schema *is* the canonical format — building our export to match it future-proofs against any filing service swap.
- **Gaps:** TCC application takes ~45 days; recipient copy delivery is on you (print + mail or email). No W-9 collection. No state filing.

### 6. Open source — moov-io/irs, sdj0/fire-1099
- **What it is:** Go (moov-io) and Python (sdj0) libraries that emit IRS FIRE-format files for 1099-NEC/MISC.
- **Pricing:** Free.
- **W-9 collection:** None.
- **Tax ID security:** None — we'd build it.
- **YTD tracking:** None.
- **E-file support:** Generates the filing *file*, doesn't transmit it.
- **UI patterns to steal:** JSON schema definitions are a clean reference for what fields a valid 1099-NEC actually needs.
- **Gaps:** FIRE is being retired after FY2026; IRIS uses a different schema. moov-io appears more actively maintained; sdj0 hasn't been touched in years.

## IRS IRIS portal — what changed in 2026+
- IRIS is now the IRS's primary direct-file portal for information returns. Supports manual entry **and** CSV upload using IRS-published templates.
- Free for any filer of any size.
- Requires a Transmitter Control Code (TCC) — apply via IRS.gov, 5-digit code, takes weeks (apply NOW for January).
- Mandatory e-file threshold: 10+ information returns aggregated across all types (W-2 + 1099 + etc.). We're under that, so paper is still legal, but IRIS makes paper pointless.
- FIRE retires after filing season 2026. Anything we build targeting FIRE format is dead-on-arrival.
- IRIS does NOT collect W-9s, does NOT mail recipient copies, does NOT handle state filing. Those are the gaps a service like Track1099 fills.

## Cross-cutting patterns
1. **Self-service W-9 link is universal.** Every serious tool sends an email link; the payer never types a SSN. Manual PDF upload is the fallback, not the norm.
2. **TIN masking everywhere.** SSN shows as `***-**-1234` in every UI we surveyed. Only the contractor (during W-9 entry) and the IRS submission see the full value.
3. **Status pills on contractor list:** W-9 Requested / W-9 Received / TIN Verified / 1099 Filed.
4. **$600 NEC threshold is universal.** Tools warn "approaching threshold" around $500 YTD. (Note: backup withholding threshold rose to $2,000 in 2026, but the 1099-NEC issuance threshold stays at $600.)
5. **TIN match before file** is a paid add-on everywhere ($0.45-$1.00). Eliminates 1099 corrections later.
6. **Recipient delivery preference order:** e-delivery (free, requires recipient consent) > postal mail (paid) > paper handoff (manual).
7. **State filing is the hidden complexity.** CF/SF program covers 32 states, but MA, MN, MI, OR still require direct state filing. **Tennessee has no income tax → no state 1099-NEC filing required for The Barrax.** This is a meaningful simplification.

## Interesting differentiators
- **TaxBandits' free W-9 vault** — they bet that collection drives filing, and give the collection layer away. Confirms W-9 collection is the high-friction step, not filing.
- **Gusto's self-onboarding link** — contractor sees a branded form, completes W-9 + payment info in one flow; the payer never sees the SSN even masked.
- **Track1099's API + embeddable widget** — implies the modern pattern is "W-9 collection as a component," not a standalone product.
- **Backup withholding automation (Tipalti, 1099Pro)** — only enterprise tools automate the 24% withhold. For 3-8 contractors, this is a policy problem, not a software problem: "no W-9, no payment."

## Recommendations for our build

### E-file ourselves or hand off to CPA?
**Hand off via CSV to the CPA.** With 3-8 contractors, we file ~3-8 1099-NECs once a year. Building IRIS submission code is real work (TCC, schema validation, ack handling) for a once-a-year operation. The CPA either already has Track1099/Tax1099 in their workflow or will gladly take a CSV.

Build target: a **CSV export matching the IRS IRIS template** (not the deprecated FIRE format). If the CPA wants Track1099 format instead, it's a column rename. Worst case, paste CSV into IRIS portal manually — $0, 10 minutes.

### Tax ID encryption approach
- TIN stored encrypted at rest using Supabase's `pgsodium`/`pgcrypto` extension OR application-level encryption with a key in Vercel env vars.
- Decryption only inside the year-end CSV export endpoint, behind staff auth + audit log entry.
- TIN never returned to the UI in full — list/detail views show `***-**-1234`. Only the export step decrypts.
- Audit log every read of full TIN (who, when, why).
- W-9 PDF stored in Supabase Storage with RLS; signed URLs only for staff role.

### W-9 collection flow
Two-step pragmatic build:
1. **MVP (week 1):** Staff uploads contractor's signed W-9 PDF, manually keys legal name, address, TIN, entity type. Confirmation screen masks TIN before save.
2. **V2 (later):** Email-link flow — contractor gets a one-time signed link, completes a web form with name/TIN/address/entity, e-signs (typed name + timestamp + IP). PDF generated server-side. SSN goes from contractor's browser straight to encrypted storage; payer never sees it.

Start with MVP. We have 5 contractors total — the email-link investment isn't worth it until churn picks up.

### YTD tracking
Roll up from existing Bills table by `contractor_id` filtered to current calendar year. Display "$X YTD — approaching $600" warning at $500. Backup-withholding flag if contractor has no W-9 on file and YTD ≥ $2,000 (the new 2026 floor).

## Suggested Linear issues
- **Contractor data model** — `contractors` table with encrypted `tax_id`, entity type, address, W-9 PDF reference, `w9_received_at`. Justification: foundation for everything else.
- **W-9 PDF upload + masked display** — staff form, RLS on storage bucket, TIN masked in all list views. Justification: lets us file the existing W-9s for Eric and Adam immediately.
- **TIN encryption + audit log** — `pgsodium` or app-level encryption, audit table records every full-TIN read. Justification: this is SSN data; we cannot ship without it.
- **Bill → contractor YTD rollup** — link existing Bills to contractor, compute YTD, surface on contractor detail. Justification: the "approaching $600" signal is the whole point.
- **Backup withholding flag (advisory only)** — boolean derived from "no W-9 + YTD ≥ $2,000." Justification: prevents accidental large payments without W-9; matches 2026 IRS threshold change.
- **Year-end IRIS CSV export** — endpoint that emits IRS IRIS-format CSV for current tax year, audit-logged. Justification: the actual hand-off deliverable to the CPA.
- **Apply for IRS IRIS TCC** — non-code task for Greg; ~45-day lead time. Justification: even if CPA files, having our own TCC is the contingency. *(Note: Greg has decided to skip this — CPA handles all filing.)*
- **Recipient delivery log** — field to record when each 1099-NEC was sent to the contractor (email or mail) and method. Justification: IRS requires recipient copy by Jan 31; we need proof we sent it.
